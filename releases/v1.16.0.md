## What's New in v1.16.0

### 📋 Control Plane Enhancements
- **License Management**: View license details and receive expiration notifications (90, 30, 7 days)
- **Hybrid Proxy**: Improved hybrid connectivity for cloud-edge integration

### 🔧 BusinessWorks Capabilities
- **BW6 AI Plugin 6.0.0** (Preview): RAG (Retrieval-Augmented Generation) Palette for AI-powered applications
- **BW5 Monitoring**: Historical Logs, Audit History, and interactive Metrics charts
- **BW5 Containers**: Custom Base Image support and CyberArk Conjur credential management
- **Domain Permissions**: Enhanced permission indicators ('Read'/'Write') on Domain cards

### 🎯 Flogo Enhancements
- ** and Sidecar Containers**: Support in `values.yaml` for Helm deployments
- **API Version Control**: Specify connector/extension versions in build API
- **Resource Customization**: Configurable provisioner resources in Helm chart
- **CLI Improvements**: `--all` flag for `list-connectors` and `list-flogo-versions` commands

### 🛠️ Developer Hub
- **URL Management**: Update Developer Hub URL through UI with ingress controller flexibility

## Upgrade Path from v1.15.0

### Breaking Changes
⚠️ **Container Registry**: Update registry URL and credentials in values files  
⚠️ **Subscriptions**: If using custom hostPrefix for subscriptions, ensure DNS records are created  

### Migration Steps
1. **Backup**: Export current v1.15.0 configuration and databases
2. **Update Values**: Change container registry URL to `csgprdusw2reposaas.jfrog.io`
3. **DNS Records**: Add `ai.dp1.atsnl-emea.azure.dataplanes.pro` A record
4. **Helm Upgrade**: Use `helm upgrade` with v1.16.0 charts
5. **Verify**: Check all pods are running and ingress routes are accessible

Detailed upgrade instructions: See [UPGRADE-1.15-TO-1.16.md](/howto/v1.16/UPGRADE-1.15-TO-1.16.md)
