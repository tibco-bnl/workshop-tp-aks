# TIBCO Platform v1.18.0 Release Notes

**Release Date**: June 2026
**TIBCO Platform Control Plane Version**: 1.18.0
**Status**: Current release

## Overview

TIBCO Platform Control Plane v1.18.0 adds Gateway API Controller support, namespace-level RBAC for Kubernetes data planes, alert audit history, Console-managed email server configuration, and Developer Hub self-service flows. This release is aligned with the 1.18.0 chart artifacts in the official [TIBCO tp-helm-charts repository](https://github.com/TIBCOSoftware/tp-helm-charts).

Official documentation:

- [Official 1.18.0 Known Issues](https://docs.tibco.com/pub/platform-cp/1.18.0/doc/html/Release-Notes/known-issues.htm)

## What's New in v1.18.0

### TIBCO Control Plane

#### Gateway API Controller Support for Control Tower Data Plane

Gateway API Controller is supported for Control Tower data planes. This is important for AKS environments that are moving from classic Kubernetes Ingress patterns toward Gateway API resources.

#### Alert Viewer and Alerts Audit Trail

The UI includes an Alerts Audit Trail page that records system health and rule performance events.

#### Namespace-Level RBAC for Data Planes

Kubernetes namespaces are now Data Plane-level resources managed under Data Plane Configuration > Resources. Application Manager and Application Viewer role assignment screens include Capability and Namespace selectors for granular access control.

#### Email Server Configuration in Platform Console

Email server configuration moved from `tibco-cp-base` chart values to the TIBCO Platform Console UI. Configure SMTP from the Console after installation or upgrade.

### BusinessWorks 6 Containers

- Traefik is supported as a Gateway API controller for BW6 Containers capabilities.
- BW6 application endpoints can be exposed publicly using Traefik Gateway API.
- An `Other` Gateway API controller type allows third-party Gateway API controllers already installed in the cluster.

### BusinessWorks 5 Containers

- Traefik is supported as a Gateway API controller for BW5 Containers capabilities.
- BW5 application endpoints can be exposed publicly using Traefik Gateway API.
- An `Other` Gateway API controller type allows third-party Gateway API controllers already installed in the cluster.
- The open source `bw5toCE` automation tool is available to help migrate BW5 Administrator applications to TIBCO Platform.

### Flogo Capability

- Traefik Gateway API support for Kubernetes data planes.
- NetScaler Gateway support at the data plane level.
- Namespace-level RBAC enforcement for Application Manager and Application Viewer roles.

### Developer Hub

- Self-service flows provide reusable building blocks for platform automation through customizable templates and platform APIs.

## AKS-Specific Considerations

### Simplified DNS Continues

The simplified DNS model remains applicable in 1.18.0. AKS installations should prefer one Control Plane base domain such as `platform.azure.example.com`, with Platform Console at `admin.platform.azure.example.com` and subscriptions at `<hostPrefix>.platform.azure.example.com`. In Helm values, set `global.external.dnsDomain` and `global.external.dnsTunnelDomain` to the same base domain; the `hybrid-proxy` chart routes tunnel traffic through `/infra/tunnel` on that shared domain. Keep separate `cp1-my` and `cp1-tunnel` domains only for legacy or explicitly separated deployments.

### Gateway API and Traefik

AKS workshops already standardize on Traefik for ingress. In 1.18.0, evaluate Traefik Gateway API for application endpoint exposure where the capability supports Gateway API. Keep classic Traefik ingress for existing routes unless you intentionally migrate endpoint exposure.

### Namespace-Level RBAC

Before upgrading shared AKS clusters, inventory application namespaces and review Application Manager/Application Viewer assignments. After upgrade, users may need explicit namespace grants to deploy or view applications.

### Email with MailDev

MailDev remains useful for workshop SMTP testing, but the SMTP configuration should be entered in the Platform Console after 1.18.0 install or upgrade.

### Upgrade Capabilities with Infrastructure

Do not upgrade only Control Plane infrastructure and leave capabilities behind. Several observability, endpoint visibility, and application-management behaviors depend on matching or current capability versions.

## Upgrade Path from v1.17.0

1. Back up Helm values and PostgreSQL data.
2. Update the `tibco-platform` Helm repository.
3. Upgrade Control Plane base and infrastructure charts.
4. Upgrade Data Plane components and capability charts.
5. Configure email server from the TIBCO Platform Console.
6. Review namespace-level RBAC assignments.
7. Validate Gateway API endpoint behavior if used.
8. Confirm Alerts Audit Trail events are recorded.

## Upgrade Path from v1.16.0

Upgrade to 1.17.0 first, then upgrade to 1.18.0:

1. Follow [v1.16.0 to v1.17.0 upgrade notes](./v1.17.0.md#upgrade-path-from-v1160).
2. Apply the v1.18.0 upgrade path above.

## Known Issues to Review

The official 1.18.0 known issues include items relevant to AKS and Traefik users:

- Existing 1.17.0 Data Planes can show incorrect Integration Summary and Application Instance card data after Control Plane upgrade until the Data Plane is upgraded to 1.18.0.
- Traces and application service metrics have known limitations with Traefik in some Control Tower data plane environments.
- BW6 Swagger tester behavior can require BW6 capability and application upgrades when changing endpoint visibility through Traefik Gateway.
- Flogo capability provisioning can fail if users navigate backward and forward in early wizard steps; restart the provisioning flow as a workaround.

Review the complete official list before production upgrades: [Official 1.18.0 Known Issues](https://docs.tibco.com/pub/platform-cp/1.18.0/doc/html/Release-Notes/known-issues.htm).
