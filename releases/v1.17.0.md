# TIBCO Platform v1.17.0 Release Notes

**Release Date**: May 18, 2026  
**TIBCO Platform Control Plane Version**: 1.17.0  
**Status**: ✅ Production Ready

---

## Overview

TIBCO Platform Control Plane v1.17.0 brings significant improvements in observability (OpenSearch support), alerting (Webhook receiver), BW6 classic lifecycle management, and Flogo extensibility. This release is aligned with the tp-helm-charts v1.17.0 release available at [https://github.com/TIBCOSoftware/tp-helm-charts](https://github.com/TIBCOSoftware/tp-helm-charts).

> **Source:** [Official TIBCO Platform 1.17.0 Release Notes](https://docs.tibco.com/pub/platform-cp/1.17.0/doc/html/Release-Notes/new-features.htm)

---

## What's New in v1.17.0

### 📋 Control Plane Enhancements

#### Webhook Receiver for Alerts
- **Standardized JSON Payload**: Integrate alert notifications with any external system via HTTP webhook
- **Flexible Integration**: Connect to PagerDuty, ServiceNow, Slack, Teams, or any custom webhook endpoint
- **Configurable per Alert Rule**: Define custom webhook endpoints globally or per alert rule
- See: [Configuring Webhook Notification Receiver](https://docs.tibco.com/pub/platform-cp/1.17.0/doc/html/UserGuide/configuring-webhook-receiver.htm)

#### OpenSearch Support for Observability
- **OpenSearch as Observability Backend**: Use OpenSearch instead of (or alongside) Elasticsearch for:
  - Jaeger distributed traces
  - Service logs from capabilities
- **Index Template Management**: Optimized OpenSearch index templates for TIBCO Platform workloads
- **Greater Flexibility**: Choose your observability storage backend based on operational requirements
- See: [OpenSearch Index Templates](https://docs.tibco.com/pub/platform-cp/1.17.0/doc/html/UserGuide/jaeger-opensearch-index-template.htm)

#### Capability Management APIs
- **Capability Update API (PUT)**: Update existing capability instances without full re-provisioning
- **Capability Upgrade API**: Upgrade capability instances programmatically — ideal for CI/CD automation pipelines

### 🔧 BusinessWorks 6 (Containers)

#### Custom Fluentbit Configuration
- **User-Defined Log Pipelines**: Set customized Fluentbit configurations while provisioning or updating the BW6 (Containers) Capability
- **Flexible Log Routing**: Tailor log parsing, filtering, and output targets to match operational requirements
- **Minimum Capability Version**: 1.17.0

### 🔧 TIBCO ActiveMatrix BusinessWorks™ 6 (Classic)

#### Complete Lifecycle Management via Control Plane UI
A major enhancement — full BW6 classic environment management is now available directly in the Control Plane UI:

**Agent Management**
- View registered BW6 Agent Networks (domains, agents, machines, installations)
- List Agents, machines, and installations with full details: status, version, OS, IP, location
- Pagination, filtering, and global refresh

**Domain Management**
- List BW6 Domains (name, description, entity counts)
- Create, backup, and delete domains

**Application Archive Management**
- Upload, Configure, Deploy, Download, and Delete Application Archives
- View archive details: deployed apps, appspaces, components, processes, deployment history

**AppSpace Management**
- Create, Update, Delete AppSpaces
- Deploy applications to AppSpaces
- Manage AppNodes, Agents, Machines, Installations, and Command History per AppSpace

**AppNode Management**
- Create, Update, Delete AppNodes
- Lifecycle operations: start/stop
- Update AppNode Properties, manage Application Instances
- Collect/download AppNode data, view Command History for audit

**Application Management**
- View, start/stop, undeploy applications
- Update application and instance properties
- Manage instances, appnodes, endpoints, components, processes
- Application Command History and Execution History for full audit trail

### 🔧 TIBCO ActiveMatrix BusinessWorks™ 5 and Adapters

#### Application History Audit Trail
- **Application Configuration History**: View application lifecycle operations (deploy/undeploy) under the new **History** tab in the Application Configuration section
- Complete audit trail for application lifecycle changes

### 🔧 BusinessWorks 5 (Containers)

#### Custom Fluentbit Configuration
- **User-Defined Log Pipelines**: Set customized Fluentbit configurations while provisioning or updating the BW5 (Containers) Capability
- **Minimum Capability Version**: 1.17.0

#### Hawk Methods REST API
- **New REST Endpoint**: Port 8090 under `/commands` in BW5CE exposes Hawk Methods via REST API
- **31 Hawk Methods Supported**: Excludes asynchronous and sensitive operations
- **Programmatic Hawk Access**: Enables REST-based access to Hawk monitoring operations

### 🎯 Flogo Enhancements

#### Fluentbit Configuration via Helm
- **Helm-Native Config**: Configure Fluentbit directly via Helm chart values for Flogo capability
- **Consistent Configuration Management**: Manage log forwarding as part of standard Helm deployment workflow
- See: [Configuring Fluent Bit for TIBCO Flogo in a Dataplane](https://docs.tibco.com/pub/platform-cp/1.17.0/doc/html/Subsystems/flogo-capability/flogo-user-guide/configuring-fluent-bit.htm)

#### Flogo Capability Recipe Customization
- **In-UI Recipe Editor**: Modify YAML configuration directly in Control Plane UI when provisioning or updating the Flogo capability
- **Resource Allocation Control**: Fine-tune CPU/memory requirements per deployment
- **Deployment Flexibility**: Handle specific deployment requirements through recipe customization
- See: [Provisioning and Upgrading TIBCO Flogo Capability](https://docs.tibco.com/pub/platform-cp/1.17.0/doc/html/Subsystems/flogo-capability/flogo-user-guide/provisioning-flogo-capability.htm)

#### Newly Supported Connectors
- **TIBCO Flogo® Connector for Google Cloud Storage**: Read/write objects in GCS buckets
- **TIBCO Flogo® Connector for TIBCO ActiveSpaces®**: Integration with TIBCO ActiveSpaces data grid
- **TIBCO Flogo® Connector for TIBCO FTL®**: Integration with TIBCO FTL messaging

## AKS-Specific Considerations

### OpenSearch vs Elasticsearch on AKS
- Both Elasticsearch (ECK) and OpenSearch can be deployed on AKS
- For new deployments, consider OpenSearch as an alternative to ECK
- Existing ECK-based observability stacks continue to work with v1.17.0
- OpenSearch can be deployed via the OpenSearch Operator for Kubernetes

### BW5CE Hawk REST API — NetworkPolicy Update
- The new port 8090 endpoint in BW5CE pods may require NetworkPolicy updates
- If you have restrictive ingress/egress policies for Data Plane namespaces, add port 8090 rules:
  ```yaml
  # Example NetworkPolicy addition for BW5CE pods
  - ports:
    - port: 8090
      protocol: TCP
  ```

### Webhook Receiver — Egress Firewall Rules
- If using the Webhook Receiver for Alerts, ensure the Control Plane namespace (`cp1-ns`) can reach the external webhook endpoint
- Update AKS NSG and NetworkPolicy egress rules as required

### Ingress Configuration (unchanged from v1.16.0)
- **Ingress Controller**: Traefik (recommended)
- **IngressClass**: `traefik`

## Upgrade Path from v1.16.0

### Steps

1. **Backup Current Installation**:
   ```bash
   # Backup all helm releases
   for release in $(helm list -n cp1-ns -o json | jq -r '.[].name'); do
     helm get values $release -n cp1-ns > ${release}-values-backup.yaml
   done
   ```

2. **Update Helm Repository**:
   ```bash
   helm repo update tibco-platform
   ```

3. **Upgrade Control Plane Base**:
   ```bash
   helm upgrade tibco-cp-base tibco-platform/tibco-cp-base \
     --version 1.17.0 \
     -n cp1-ns \
     -f cp1-values.yaml
   ```

4. **Configure OpenSearch (optional — if switching from Elasticsearch)**:
   ```bash
   # Apply OpenSearch index templates
  # See: https://docs.tibco.com/pub/platform-cp/1.17.0/doc/html/UserGuide/jaeger-opensearch-index-template.htm
   ```

5. **Verify Upgrade**:
   ```bash
   kubectl get pods -n cp1-ns
   helm list -n cp1-ns
   kubectl get ingress -n cp1-ns
   ```

### From v1.15.0 to v1.17.0

**Not recommended** — upgrade to v1.16.0 first, then to v1.17.0:
1. Follow [v1.15.0 → v1.16.0 upgrade path](./v1.16.0#upgrade-path-from-v1150)
2. Then follow v1.16.0 → v1.17.0 steps above

## Known Issues & Workarounds

### OpenSearch Integration
- **Note**: When using OpenSearch for observability, apply the recommended index templates before connecting TIBCO Platform
- **Reference**: [OpenSearch Index Templates](https://docs.tibco.com/pub/platform-cp/1.17.0/doc/html/UserGuide/jaeger-opensearch-index-template.htm)

### BW5CE Port 8090
- **Note**: Port 8090 is a new port exposed by BW5CE pods for the Hawk REST API
- **Action**: Update NetworkPolicy rules for Data Plane namespaces if you have restrictive policies

### Helm Release Status
- **Issue**: Helm release may show "failed" status after upgrade if it times out
- **Impact**: None — pods remain operational despite helm status display
- **Workaround**: Components are fully operational; use `kubectl get pods -n cp1-ns` to verify
